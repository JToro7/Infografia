Infografía: Introducción a los Sistemas de Gestión de Bases de Datos (SGDB)

1. Definición

Un SGDB es un software que permite la creación, manipulación y gestión de bases de datos, facilitando el almacenamiento, organización y acceso a la información de manera eficiente.

2. Historia

  • 1930s: Inicio de las primeras computadoras y almacenamiento digital básico.
  • 1963: Primer uso del término "bases de datos".
  • 1970: Edgar Codd introduce el modelo relacional.
  • 1970s-1990s: Surgimiento de sistemas como Oracle, SQL y Microsoft Access.

3. Características de un SGDB

  • Organización estructurada: Usa tablas para gestionar datos.
  • Reducción de redundancia: Evita la duplicación de información.
  • Integridad de los datos: Garantiza datos precisos y consistentes.
  • Seguridad: Control de acceso y privilegios.
  • Consulta y recuperación: Acceso y manipulación de datos rápidos.

4. Componentes Principales

  • Tablas: Estructuras de almacenamiento de datos.
  • Campos: Atributos de una entidad.
  • Registros: Conjuntos de datos completos.
  • DBA: Administrador responsable de la seguridad y operación.

5. Arquitectura

  • Cliente-Servidor: Modelo donde el servidor aloja la base de datos.
  • Monolítica: Todo en un único sistema.
  • Distribuida: Bases de datos repartidas en múltiples nodos.

6. Sistemas de Gestión de Bases de Datos más Conocidos

Oracle: Estable y con soporte para transacciones complejas.
Microsoft SQL Server: Integración con el ecosistema de Microsoft.
MySQL: Popular en aplicaciones web, de código abierto.
Informix: Eficiencia en grandes volúmenes de datos.
DB2: Usado por IBM en aplicaciones empresariales de gran escala.

7. Beneficios de usar un SGDB

  • Acceso rápido: Consultas optimizadas.
  • Control centralizado: Simplifica la administración de datos.
  • Escalabilidad: Adaptable a grandes volúmenes.
  • Seguridad mejorada: Roles y permisos de acceso.
